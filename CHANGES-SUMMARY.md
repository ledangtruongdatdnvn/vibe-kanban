# Change Summary

Snapshot based on `upstream/main...main` after running
`./update-changes-tree.sh`.

| Type | Area | Added feature or fixed problem | Key code |
| --- | --- | --- | --- |
| Added | Host Admin stack | New standalone Host Admin service and React app protected by a shared secret and cookie session. It gives operators a separate surface for credentials, workspaces, branches, cleanup, and repo terminal access. | `services/admin/server.js`, `packages/admin/src/`, `services/admin/Dockerfile`, `docker-compose.yaml` |
| Added | Credentials management | Claude and Codex credentials can be pasted into the admin UI, saved into mounted Docker volumes, inspected, and cleared. This removes the need for interactive auth inside the host container. | `services/admin/server.js`, `packages/admin/src/features/admin/ui/CredentialsSection.tsx` |
| Added | Workspace visibility and cleanup | New admin endpoints compute workspace disk usage, repo worktree sizes, and orphan cleanup results. The UI can trigger orphan cleanup and persisted-host-data wipes from one place. | `crates/server/src/routes/admin.rs`, `packages/admin/src/features/admin/ui/WorkspacesSection.tsx`, `packages/admin/src/features/admin/ui/CleanupSection.tsx` |
| Added | Safe branch lifecycle | Host Admin can list branches and delete only local, non-current branches. The backend now blocks deletion when a workspace still references the branch and returns the conflicting workspace names. | `crates/server/src/routes/repo.rs`, `packages/admin/src/features/admin/ui/BranchesSection.tsx` |
| Added | Repo terminal with managed Git auth | New repo terminal WebSocket opens a shell at the repo root and injects GitHub App HTTPS auth into that shell, including SSH-to-HTTPS rewrite plus an askpass helper for short-lived tokens. | `crates/server/src/routes/terminal.rs`, `crates/server/src/repo_git_auth.rs`, `crates/local-deployment/src/pty.rs`, `packages/admin/src/features/admin/ui/TerminalSection.tsx` |
| Fixed | Repo clone reliability | GitHub imports now use native `git clone` with an HTTP auth header instead of libgit2 credential callbacks. This avoids the HTTPS/TLS/auth failures that often happen in container deployments. | `crates/git/src/cli.rs`, `crates/git/src/lib.rs` |
| Added | Host-side GitHub repo import | Repositories can now be imported from GitHub by `owner/repo`, HTTPS URL, or SSH URL, cloned into the managed host repo root, validated against existing remotes, registered automatically, and given default metadata. | `crates/services/src/services/repo.rs`, `crates/server/src/routes/repo.rs`, `packages/admin/src/features/admin/ui/TerminalSection.tsx`, `packages/web-core/src/shared/dialogs/shared/ImportGitHubRepoDialog.tsx`, `packages/web-core/src/shared/components/CreateModeRepoPickerBar.tsx` |
| Added | GitHub App organization management | Remote org settings now support installing or adopting existing GitHub App installations, syncing granted repositories, removing the local link, and toggling PR review automation per repo or in bulk. | `crates/remote/src/routes/github_app.rs`, `crates/remote/src/github_app/service.rs`, `packages/web-core/src/shared/dialogs/settings/settings/OrganizationsSettingsSection.tsx`, `packages/remote-web/src/pages/HomePage.tsx`, `packages/remote-web/src/routes/account_.organizations.$orgId.tsx` |
| Added | Short-lived repo access tokens | The remote backend can now resolve the GitHub App installation for a repository and mint a repo-scoped installation token for members of the linked organization. Host import and terminal flows reuse that token. | `crates/api-types/src/github_app.rs`, `crates/remote/src/db/github_app.rs`, `crates/services/src/services/remote_client.rs`, `crates/server/src/repo_git_auth.rs` |
| Fixed | Admin terminal proxy | The admin proxy now handles WebSocket upgrade failures cleanly and stops forwarding the browser `Origin` header on the server-to-server hop, which reduces false terminal upgrade rejections. | `services/admin/server.js` |
| Fixed | Frontend and build polish | Added a reusable copy button, switched the changes panel to the portable diff worker, and wired the new admin package into repo `check` and `format` flows plus Tailwind scanning so the new UI builds consistently. | `packages/ui/src/components/CopyButton.tsx`, `packages/web-core/src/shared/components/CopyButton.tsx`, `packages/web-core/src/pages/workspaces/ChangesPanelContainer.tsx`, `package.json`, `packages/local-web/tailwind.new.config.js` |
| Added | Self-host deployment support | Docker Compose now includes Azurite, host/admin wiring, host ownership bootstrap, and docs/scripts for deployment and destructive data wipes. This makes the self-hosted stack more complete to operate. | `docker-compose.yaml`, `crates/server/Dockerfile`, `crates/server/docker-entrypoint.sh`, `DEPLOYMENT.md`, `PRODUCTION-REMOVE-DATA.md`, `local-remove-data.sh` |
